\doxysubsubsection{Detailed Description}
\subsubsection*{template$<$int D, typename T$>$\newline
class Pscf\+::\+Vec$<$ D, T $>$}

A Vec$<$\+D, T$>$$<$D,T$>$ is a D-\/component vector with elements of type T. 

The elements of a Vec$<$\+D, T$>$ can be accessed using subscript operator, as for a built in array.

The arithmetic assignment operators +=, -\/=, and $\ast$= are overloaded to allow vector-\/vector addition and subtraction and vector-\/scalar multiplication.

All other unary and binary mathematical operations are implemented as methods or free functions. Operations that yield a Vec$<$\+D, T$>$, such as addition, are implemented by methods that assign the result to the invoking \mbox{\hyperlink{classPscf_1_1Vec}{Vec}} object, and return this object by reference. For example, 
\begin{DoxyCode}{0}
\DoxyCodeLine{Vec<3, double> a, b, c;}
\DoxyCodeLine{}
\DoxyCodeLine{a[0] = 0.0}
\DoxyCodeLine{a[1] = 1.0}
\DoxyCodeLine{a[2] = 2.0}
\DoxyCodeLine{}
\DoxyCodeLine{b[0] =  0.5}
\DoxyCodeLine{b[1] = -\/0.5}
\DoxyCodeLine{b[2] = -\/1.5}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{// Set a = a + b}}
\DoxyCodeLine{a += b}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{// Set b = b*2}}
\DoxyCodeLine{b *= 2.0;}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{// Set c = a + b}}
\DoxyCodeLine{c.add(a, b);}
\end{DoxyCode}
 This syntax for functions that yield a vector makes the allocation of a temporary Vec$<$\+D, T$>$ object explicit, by requiring that the invoking function be a member of an object that will hold the result.

For efficiency, all member functions are declared inline.
