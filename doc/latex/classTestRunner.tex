\doxysubsubsection{Detailed Description}
Abstract base class for classes that run tests. 

\mbox{\hyperlink{classTestRunner}{Test\+Runner}} is an abstract base class with two types of subclass\+: The \mbox{\hyperlink{classUnitTestRunner}{Unit\+Test\+Runner}} class template defines a \mbox{\hyperlink{classTestRunner}{Test\+Runner}} that runs the tests for an associated \mbox{\hyperlink{classUnitTest}{Unit\+Test}}. A \mbox{\hyperlink{classCompositeTestRunner}{Composite\+Test\+Runner}} runs the tests for a sequence of other \mbox{\hyperlink{classTestRunner}{Test\+Runner}} objects, each of which can be a \mbox{\hyperlink{classUnitTestRunner}{Unit\+Test\+Runner}} or another \mbox{\hyperlink{classCompositeTestRunner}{Composite\+Test\+Runner}}.

An implementation of the pure virtual \mbox{\hyperlink{classTestRunner_a3216b1149a6e1f207253c1659a617e77}{run()}} method of must run all of the associated test methods, and records the number \mbox{\hyperlink{classTestRunner_a6b8aee43d4dba17affc7d0c79569fbca}{n\+Success()}} of tests that succeed and the number \mbox{\hyperlink{classTestRunner_ad16612e6afc78a84bb970e6c9dcc7a6a}{n\+Failure()}} that fail. A test fails if it throws a \mbox{\hyperlink{classTestException}{Test\+Exception}}. Test methods use the T\+E\+S\+T\+\_\+\+A\+S\+S\+E\+R\+T(expr) macro to assert the truth of a logical exprression expr, which throws a \mbox{\hyperlink{classTestException}{Test\+Exception}} if expr is false. The implementation of \mbox{\hyperlink{classTestRunner_a3216b1149a6e1f207253c1659a617e77}{run()}} for a \mbox{\hyperlink{classUnitTestRunner}{Unit\+Test\+Runner}} runs each unit test method of the associated \mbox{\hyperlink{classUnitTest}{Unit\+Test}} in a try-\/catch block and catches any thrown Test\+Exceptions. The implementation of run for a Test\+Composite calls the \mbox{\hyperlink{classTestRunner_a3216b1149a6e1f207253c1659a617e77}{run()}} method for each of its children.

Each \mbox{\hyperlink{classTestRunner}{Test\+Runner}} may optionally have a parent \mbox{\hyperlink{classTestRunner}{Test\+Runner}}. The parent if any, is always a Test\+Composite. A Test\+Composite can have any number of children. ~\newline


The \mbox{\hyperlink{classTestRunner_ab5810788fc836edbc8285d3ad2f7c2bd}{record\+Failure()}} and \mbox{\hyperlink{classTestRunner_acc6fe0c6afd26fd05dcdc8e5addb193d}{record\+Success()}} methods of a \mbox{\hyperlink{classTestRunner}{Test\+Runner}}, which can be called by the run method, increment the n\+Success or n\+Failure counters. If the \mbox{\hyperlink{classTestRunner}{Test\+Runner}} has a parent, each function also calls the corresponding function of the parent, thus incrementing the corresponding counter of the parent. The n\+Success and n\+Failure counters for a Test\+Composite thereby keep track of the total number of successful and failed unit test methods run by all descendants.

Each \mbox{\hyperlink{classTestRunner}{Test\+Runner}} has a file\+Prefix string. The file\+Prefix is initialized to an empty string, and may be modified by the virtual \mbox{\hyperlink{classTestRunner_ae977948edaf2121faa85af96b7aefdcf}{add\+File\+Prefix()}} function. The default implementation of this function prepends a string argument to the existing file\+Prefix.

The \mbox{\hyperlink{classUnitTestRunner}{Unit\+Test\+Runner}} class template supplies the file\+Prefix to instances of the associated \mbox{\hyperlink{classUnitTest}{Unit\+Test}} class when a \mbox{\hyperlink{classUnitTest}{Unit\+Test}} is created. (See the notes for \mbox{\hyperlink{classUnitTestRunner}{Unit\+Test\+Runner}} for details of how). The file\+Prefix string of the \mbox{\hyperlink{classUnitTest}{Unit\+Test}} is then prepended to the names of any files opened by the functions open\+Input\+File(), open\+Output\+File(), and open\+File of the \mbox{\hyperlink{classUnitTest}{Unit\+Test}} subclass.

The implementation of \mbox{\hyperlink{classTestRunner_ae977948edaf2121faa85af96b7aefdcf}{add\+File\+Prefix()}} by the Test\+Composite subclass calls the add\+File\+Prefix method of each of its children, and thus allows a common prefix to be added to the file paths used by all of its children.
