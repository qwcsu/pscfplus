\doxysubsubsection{Detailed Description}
\mbox{\hyperlink{classUnitTest}{Unit\+Test}} is a base class for classes that define unit tests. 

Each subclass of \mbox{\hyperlink{classUnitTest}{Unit\+Test}} should define one or more test methods. Each test method must be a zero parameter function that returns void. Test methods may be given arbitrary names. Individual test methods should use the preprocessor macro T\+E\+S\+T\+\_\+\+A\+S\+S\+E\+R\+T(expression) defined in Text\+Exception.\+h to assert the truth of logical expressions.

The test methods defined by a \mbox{\hyperlink{classUnitTest}{Unit\+Test}} are run by an associated subclass of \mbox{\hyperlink{classTestRunner}{Test\+Runner}}. Each test method of a \mbox{\hyperlink{classUnitTest}{Unit\+Test}} must be added to the associated \mbox{\hyperlink{classTestRunner}{Test\+Runner}}. The run() method of a \mbox{\hyperlink{classTestRunner}{Test\+Runner}} calls all of the associated test methods in the order in which they were added, and counts the number of successful and failed tests.

The \mbox{\hyperlink{classTestRunner}{Test\+Runner}} associated with a single \mbox{\hyperlink{classUnitTest}{Unit\+Test}} is defined by a class template \mbox{\hyperlink{classUnitTestRunner}{Unit\+Test\+Runner}}, which takes a \mbox{\hyperlink{classUnitTest}{Unit\+Test}} subclass as a template argument. For example, the \mbox{\hyperlink{classTestRunner}{Test\+Runner}} associated with a \mbox{\hyperlink{classUnitTest}{Unit\+Test}} subclass named \mbox{\hyperlink{classTestA}{TestA}} is a template instantiation Unit\+Test\+Runner$<$\+Test\+A$>$.

Preprocessor macros defined in the file \mbox{\hyperlink{UnitTestRunner_8h_source}{Unit\+Test\+Runner.\+h}} should be used to create the boiler-\/plate code necessary to define a unit test runner and to add test methods to it.
