\doxysubsubsection{Detailed Description}
An object that can read multiple parameters from file. 

Any class that reads a block of parameters from a parameter file must be derived from \mbox{\hyperlink{classUtil_1_1ParamComposite}{Param\+Composite}}. Each such class must implement either the \mbox{\hyperlink{classUtil_1_1ParamComposite_a172a3a656b5829c8d758424ad09a22a6}{read\+Parameters()}} function or the \mbox{\hyperlink{classUtil_1_1ParamComposite_a6fc094b057fd9a21641b089f6f063b39}{read\+Param()}} function, but not both. The \mbox{\hyperlink{classUtil_1_1ParamComposite_a172a3a656b5829c8d758424ad09a22a6}{read\+Parameters()}}, if reimplemented, should read the body of the associated parameter file block, without opening or closing lines. The \mbox{\hyperlink{classUtil_1_1ParamComposite_a6fc094b057fd9a21641b089f6f063b39}{read\+Param()}} function reads the the entire block, including opening line and closing lines. The default implementation of \mbox{\hyperlink{classUtil_1_1ParamComposite_a6fc094b057fd9a21641b089f6f063b39}{read\+Param()}} reads the opening line of the block, calls \mbox{\hyperlink{classUtil_1_1ParamComposite_a172a3a656b5829c8d758424ad09a22a6}{read\+Parameters()}} to read the body of the block, and then reads the closing line. Most subclasses of \mbox{\hyperlink{classUtil_1_1ParamComposite}{Param\+Composite}} re-\/implement the \mbox{\hyperlink{classUtil_1_1ParamComposite_a172a3a656b5829c8d758424ad09a22a6}{read\+Parameters()}} function, and rely on the default implementation of \mbox{\hyperlink{classUtil_1_1ParamComposite_a6fc094b057fd9a21641b089f6f063b39}{read\+Param()}} to add the \mbox{\hyperlink{classUtil_1_1Begin}{Begin}} and \mbox{\hyperlink{classUtil_1_1End}{End}} lines.

The \mbox{\hyperlink{classUtil_1_1ParamComposite_addf0c36c3560471bfe47f0be008575b9}{write\+Param()}} function, if called after \mbox{\hyperlink{classUtil_1_1ParamComposite_a6fc094b057fd9a21641b089f6f063b39}{read\+Param()}}, writes the associated parameter block using the same file format as that used to read the data in the earlier call to \mbox{\hyperlink{classUtil_1_1ParamComposite_a6fc094b057fd9a21641b089f6f063b39}{read\+Param()}}.\hypertarget{classUtil_1_1ParamComposite_autotoc_md0}{}\doxyparagraph{Implementation details\+:}\label{classUtil_1_1ParamComposite_autotoc_md0}
After parameter file block is read from file, the file format is stored as a private array of Para\+Component$\ast$ pointers. We will refer to this in what follows as the format\+\_\+ array. Each pointer in this array may point to a \mbox{\hyperlink{classUtil_1_1Parameter}{Parameter}}, \mbox{\hyperlink{classUtil_1_1ParamComposite}{Param\+Composite}}, \mbox{\hyperlink{classUtil_1_1Begin}{Begin}}, \mbox{\hyperlink{classUtil_1_1End}{End}}, or \mbox{\hyperlink{classUtil_1_1Blank}{Blank}} object. Pointers to these objects are added to the format array as the associated objects are read from file, and are stored in the same order as they appear in the parameter file. The default implementation of the \mbox{\hyperlink{classUtil_1_1ParamComposite_addf0c36c3560471bfe47f0be008575b9}{write\+Param()}} function simply calls the \mbox{\hyperlink{classUtil_1_1ParamComposite_addf0c36c3560471bfe47f0be008575b9}{write\+Param()}} function of each child \mbox{\hyperlink{classUtil_1_1ParamComponent}{Param\+Component}}.\hypertarget{classUtil_1_1ParamComposite_autotoc_md1}{}\doxyparagraph{Subclass implementation details\+:}\label{classUtil_1_1ParamComposite_autotoc_md1}
The \mbox{\hyperlink{classUtil_1_1ParamComposite_a172a3a656b5829c8d758424ad09a22a6}{read\+Parameters()}} function of each subclass of \mbox{\hyperlink{classUtil_1_1ParamComposite}{Param\+Composite}} should be implemented using protected member functions of \mbox{\hyperlink{classUtil_1_1ParamComposite}{Param\+Composite}} with names that begin with \char`\"{}read\char`\"{}. The \mbox{\hyperlink{classUtil_1_1ParamComposite_a21c4ae1412ec944d4189cb7c7b6a3171}{read$<$\+T$>$()}} function template can be used to read an individual parameter, while read\+Param\+Composite reads the nested subblock associated with a child \mbox{\hyperlink{classUtil_1_1ParamComposite}{Param\+Composite}}. There are also more specialized methods (e.\+g., read\+D\+Array$<$\+T$>$to read different types of arrays and matrices of parameters, and to read optional parameters. See the users manual for further details.

The \mbox{\hyperlink{classUtil_1_1ParamComposite_a30752567fd7b8b93e5c8476bbe8592c5}{set\+Class\+Name()}} and \mbox{\hyperlink{classUtil_1_1ParamComposite_a93e3a9fd3bd01dcc3c606a7bf3112c0e}{class\+Name()}} functions may be used to set and get a std\+::string containing the subclass name. The \mbox{\hyperlink{classUtil_1_1ParamComposite_a30752567fd7b8b93e5c8476bbe8592c5}{set\+Class\+Name()}} function should be called in the constructor of each subclass of \mbox{\hyperlink{classUtil_1_1ParamComposite}{Param\+Composite}}. The class name set in the constructor of a subclass will replace any name set by a base class, because of the order in which constructors are called. The default implementation of \mbox{\hyperlink{classUtil_1_1ParamComposite_a6fc094b057fd9a21641b089f6f063b39}{Param\+Composite\+::read\+Param()}} checks if the class name that appears in the opening line of a parameter block agrees with the class name returned by the \mbox{\hyperlink{classUtil_1_1ParamComposite_a93e3a9fd3bd01dcc3c606a7bf3112c0e}{class\+Name()}} function, and throws an exception if it does not.
