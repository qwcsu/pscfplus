\doxysubsubsection{Detailed Description}
A single variable in a parameter file. 

\mbox{\hyperlink{classUtil_1_1Parameter}{Parameter}} is a base class for objects that read and write the value of a single C++ variable from or to a parameter file. The parameter file format for a parameter contains a string label followed by a value for the variable. Different subclasses of parameter are specialized for different variable types, which can include primitive C/\+C++ variables, user defined types that overload the $<$$<$ and $>$$>$ operators, or any of several different types of container.

A \mbox{\hyperlink{classUtil_1_1Parameter}{Parameter}} may be required or optional element in a parameter file, depending on the value of the bool is\+Required parameter of the constructor. An optional element becomes \char`\"{}active\char`\"{} when an entry with the correct label is read from a parameter file, or when an active value is loaded from an archive. By convention, a required \mbox{\hyperlink{classUtil_1_1Parameter}{Parameter}} is always active, even before its value is read or loaded. The bool functions \mbox{\hyperlink{classUtil_1_1Parameter_accdc4c9c774b9394b387277543c1ba74}{is\+Required()}} and \mbox{\hyperlink{classUtil_1_1Parameter_ae8c6d89210cc4d3eb3eb4eb8dd3e0ff9}{is\+Active()}} can be used to query the state of a \mbox{\hyperlink{classUtil_1_1Parameter}{Parameter}}.

The overloaded \mbox{\hyperlink{classUtil_1_1Parameter_ae33aacac251a07bbc7cc98cce9dbed80}{save\+Optional()}} static member functions can be used to save optional parameters to an archive in a form that records whether or not they are active.

Read a label and (if the label matches) a parameter value. 

The parameter file format for a \mbox{\hyperlink{classUtil_1_1Parameter}{Parameter}} consists of a label string followed by value. The value is read if and only if the label matches the expected value for this \mbox{\hyperlink{classUtil_1_1Parameter}{Parameter}}. If this \mbox{\hyperlink{classUtil_1_1Parameter}{Parameter}} is required and the input label not match, an error message is printed to the log file and \mbox{\hyperlink{classUtil_1_1Exception}{Exception}} is thrown. If the \mbox{\hyperlink{classUtil_1_1Parameter}{Parameter}} is not required and the input label does not match, the label string is retained in an buffer for later processing by the read\+Param method of other \mbox{\hyperlink{classUtil_1_1ParamComponent}{Param\+Component}} objects.

Upon entry to this function, a label string is read into a label buffer if and only if the buffer is empty. This buffer is a static member of the \mbox{\hyperlink{classUtil_1_1Label}{Label}} class, which can retain a label between invocations of the read\+Parameter method of different \mbox{\hyperlink{classUtil_1_1ParamComponent}{Param\+Component}} objects. Once a label string is read from file, it remains in the label buffer until until it is matched, at which point the buffer is cleared to allow processing of the next label.
