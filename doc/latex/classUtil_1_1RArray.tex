\doxysubsubsection{Detailed Description}
\subsubsection*{template$<$typename Data$>$\newline
class Util\+::\+R\+Array$<$ Data $>$}

An \mbox{\hyperlink{classUtil_1_1Array}{Array}} that acts as a reference to another \mbox{\hyperlink{classUtil_1_1Array}{Array}} or C array. 

An \mbox{\hyperlink{classUtil_1_1RArray}{R\+Array}} is associated with a \char`\"{}target\char`\"{} \mbox{\hyperlink{classUtil_1_1DArray}{D\+Array}} or C array by the \mbox{\hyperlink{classUtil_1_1RArray_a5ce7d0c2e141487a265591cc8e6d00dc}{associate()}} method. The \mbox{\hyperlink{classUtil_1_1RArray}{R\+Array}} and its target array then wrap the same underlying C array, and so access the same data. The \mbox{\hyperlink{classUtil_1_1RArray_a5ce7d0c2e141487a265591cc8e6d00dc}{associate()}} method simply copies the address and capacity of a C array. An \mbox{\hyperlink{classUtil_1_1RArray}{R\+Array}} can be associated only once, after which it can be safely used as an alias for its target.

An \mbox{\hyperlink{classUtil_1_1RArray}{R\+Array}} can only be associated with a \mbox{\hyperlink{classUtil_1_1DArray}{D\+Array}} after the target \mbox{\hyperlink{classUtil_1_1DArray}{D\+Array}} has been allocated. Because a \mbox{\hyperlink{classUtil_1_1DArray}{D\+Array}} can be allocated only once, this association cannot be corrupted by re-\/allocation or re-\/sizing of the target \mbox{\hyperlink{classUtil_1_1DArray}{D\+Array}}.

An \mbox{\hyperlink{classUtil_1_1RArray}{R\+Array}} can be created from another \mbox{\hyperlink{classUtil_1_1RArray}{R\+Array}} only after the target \mbox{\hyperlink{classUtil_1_1RArray}{R\+Array}} has already been associated with some other \mbox{\hyperlink{classUtil_1_1Array}{Array}}.

An \mbox{\hyperlink{classUtil_1_1RArray}{R\+Array}} differs from a C++ reference to an \mbox{\hyperlink{classUtil_1_1Array}{Array}} because a C++ reference must be initialized when it is instantiated, whereas an \mbox{\hyperlink{classUtil_1_1RArray}{R\+Array}} is associated after it is instantiated. Because association is implemented by copying the address and capacity of a shared C array, access through an \mbox{\hyperlink{classUtil_1_1RArray}{R\+Array}} should be exactly as efficient as access through a \mbox{\hyperlink{classUtil_1_1DArray}{D\+Array}}.
