\doxysubsubsection{Detailed Description}
\subsubsection*{template$<$typename Data, int Capacity$>$\newline
class Util\+::\+S\+Set$<$ Data, Capacity $>$}

Statically allocated array of pointers to an unordered set. 

An \mbox{\hyperlink{classUtil_1_1SSet}{S\+Set}} is a statically allocated array that holds pointers to a set of objects. It implements the same interface as \mbox{\hyperlink{classUtil_1_1PArray}{P\+Array}} and \mbox{\hyperlink{classUtil_1_1FPArray}{F\+P\+Array}}, plus additional \mbox{\hyperlink{classUtil_1_1SSet_ad4d8da7889987726d40e003cde2d6684}{remove()}} and \mbox{\hyperlink{classUtil_1_1SSet_ad188453075d8672cb37e9b93c565e695}{index()}} methods. As for any pointer array container, the \mbox{[}\mbox{]} operator returns an associated object by reference .

An \mbox{\hyperlink{classUtil_1_1SSet}{S\+Set}} holds a set of pointers in a contiguous array. The size is the number of pointers now in the container, and the Capacity is the maximum number it can hold. The class is implemented as a wrapper for a statically allocated C array of Capacity elements.

The append method adds a pointer to the end of the sequence. The remove method removes an object from the set, or throws an exception if the object is not found in the set. As for an \mbox{\hyperlink{classUtil_1_1ArraySet}{Array\+Set}}, the remove method repacks the sequence of pointers by moving the last element to the position of the element that is being removed. Removal of an element thus generally changes the order in which the remaining elements are stored.
