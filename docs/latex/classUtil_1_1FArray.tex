\subsubsection*{template$<$typename Data, int Capacity$>$\newline
class Util\+::\+F\+Array$<$ Data, Capacity $>$}

A fixed size (static) contiguous array template. 

An \mbox{\hyperlink{classUtil_1_1FArray}{F\+Array}} is a simple wraper for a fixed size C \mbox{\hyperlink{classUtil_1_1Array}{Array}}, with a capacity that is fixed at compile time. As in a C \mbox{\hyperlink{classUtil_1_1Array}{Array}}, or a \mbox{\hyperlink{classUtil_1_1DArray}{D\+Array}} container, all of the elements are accessible. Unlike an \mbox{\hyperlink{classUtil_1_1FSArray}{F\+S\+Array}}, an \mbox{\hyperlink{classUtil_1_1FArray}{F\+Array}} does not have logical size that is distinct from its physical capacity.

When bounds checking is on (i.\+e., when N\+D\+E\+B\+UG is not defined), the operator \mbox{[}\mbox{]} checks that the index is non-\/negative and less than the Capacity.

Advice\+: Use an \mbox{\hyperlink{classUtil_1_1FArray}{F\+Array}} if you know exactly how many elements will be needed at compile time. Use an \mbox{\hyperlink{classUtil_1_1FSArray}{F\+S\+Array}} when you need a small statically allocated array for which the maximum capacity needed is known at compile time, but the logical size may be less than the capacity. Use a \mbox{\hyperlink{classUtil_1_1DArray}{D\+Array}} if you need a large, dynamically allocated array that must be allocated after instantiation.
