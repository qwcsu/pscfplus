\doxysubsubsection{Detailed Description}
\subsubsection*{template$<$typename Data, int Capacity$>$\newline
class Util\+::\+F\+P\+Array$<$ Data, Capacity $>$}

Statically allocated pointer array. 

A \mbox{\hyperlink{classUtil_1_1FPArray}{F\+P\+Array}} is a statically allocated array that actually holds pointers to Data objects, but for which the \mbox{[}\mbox{]} operator returns a reference to the associated object. It is implemented as a wrapper for a statically allocated C array of Data$\ast$ pointers. A \mbox{\hyperlink{classUtil_1_1FPArray}{F\+P\+Array}} is not responsible for destroying the associated Data objects.

The interface of an \mbox{\hyperlink{classUtil_1_1FPArray}{F\+P\+Array}} is identical to that of an \mbox{\hyperlink{classUtil_1_1FSArray}{F\+S\+Array}}. An \mbox{\hyperlink{classUtil_1_1FPArray}{F\+P\+Array}} has both a capacity that is set at compile time, which is the physical size of the underlying C array, and a size, which is the number of contiguous elements (indexed from 0 to size-\/1) that contain valid pointers. The size can only be increased only by the \mbox{\hyperlink{classUtil_1_1FPArray_a92c2e3541b37d351d5e94d84ce681c46}{append()}} method, which adds an element to the end of the array.

When compiled in debug mode, the operator \mbox{[}\mbox{]} checks that the index is less than the size and non-\/negative.
