\doxysubsubsection{Detailed Description}
\subsubsection*{template$<$typename T$>$\newline
class Util\+::\+Flex\+Ptr$<$ T $>$}

A pointer that may or may not own the object to which it points. 

\mbox{\hyperlink{classUtil_1_1FlexPtr}{Flex\+Ptr}} overloads $\ast$ and -\/$>$, and thus mimics a built-\/in pointer in most respects.

The \mbox{\hyperlink{classUtil_1_1FlexPtr_a3ede6935e7c3e80570b4cfb19879f427}{acquire(\+T$\ast$)}} method copies a built-\/in pointer and accept ownership of the object to which it points, i.\+e., accepts responsibility for deleting the object, normally when the \mbox{\hyperlink{classUtil_1_1FlexPtr}{Flex\+Ptr}} goes out of scope.

The \mbox{\hyperlink{classUtil_1_1FlexPtr_a3022ce5f92ebb70bced6853c71f2b862}{copy(\+T$\ast$)}} method copies a built-\/in pointer without accepting ownership, i.\+e., without accepting responsibility for deleting the object to which it points.

Both \mbox{\hyperlink{classUtil_1_1FlexPtr_a3ede6935e7c3e80570b4cfb19879f427}{acquire()}} and \mbox{\hyperlink{classUtil_1_1FlexPtr_a3022ce5f92ebb70bced6853c71f2b862}{copy()}} destroy any object that is already owned by this \mbox{\hyperlink{classUtil_1_1FlexPtr}{Flex\+Ptr}} before copying of a new pointer.
