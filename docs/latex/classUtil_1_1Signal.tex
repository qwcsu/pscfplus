class Util\+::\+Signal$<$ T $>$}

\mbox{\hyperlink{classUtil_1_1Notifier}{Notifier}} (or subject) in the \mbox{\hyperlink{classUtil_1_1Observer}{Observer}} design pattern. 

A \mbox{\hyperlink{classUtil_1_1Signal}{Signal}} manages a list of registered functor objects, and provides a void \mbox{\hyperlink{classUtil_1_1Signal_ae4c8adf7cb5ed88454aa2a2ca19d330b}{Signal$<$\+T$>$\+::notify(const T\&)}} method that calls them all with the same argument.

The explicit specialization \mbox{\hyperlink{classUtil_1_1Signal_3_01void_01_4}{Signal$<$void$>$}}, or Signal$<$$>$, has a notify method void \mbox{\hyperlink{classUtil_1_1Signal_ae4c8adf7cb5ed88454aa2a2ca19d330b}{Signal$<$$>$\+::notify()}} that takes no parameters, which calls a method of each observer that takes no parameters.
